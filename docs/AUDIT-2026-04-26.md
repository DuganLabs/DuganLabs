## 4. ralph-station ã `/Users/dugan/GitHub/DuganLabs/ralph-station`

### 4.1 What it is

Self-contained AI dev environment for RK3588 ARM64 single-board computers. Ships local Gemma 4 inference (Ollama) + a code editor (BaseNative IDE) + a chat UI (Open WebUI) + an autonomous coding agent (Ralph) as a single `docker compose up -d`. Three services: `ollama` (model server), `ralph-station` (Alpine + Node 22 + s6-overlay running the IDE server `ide/server.js` ã Hono + node-pty + WebSocket), `open-webui` (chat UI + complexity-routing pipeline), `caddy` (reverse proxy on port 80). Optional cloud routing to OpenRouter when complexity > threshold. Hardware-targeted: 8 GB or 16 GB RK3588 boards, with a forthcoming PCB (`docs/` "coming soon" per README). This is fundamentally a *distribution* ã not a SaaS ã and the deployment story is `git clone && docker compose up`.

### 4.2 Tech stack

- Framework / runtime: Hono 4.6 (IDE server), React 18 + Vite 6 (station-config UI), Alpine 3.21 + Node 22 + s6-overlay 3.2 (container)
- DB: none (filesystem state + Ollama model files)
- Auth: none (local-network appliance)
- Hosting: self-hosted on user's RK3588 box; no cloud component except optional OpenRouter
- Build: Vite 6 (`station-config/`), multi-stage Dockerfile (3 stages: s6 fetch ã node deps ã final image)
- Package manager: npm (per `package-lock.json*` glob in Dockerfile)
- Test: none
- Lint/format: none

### 4.3 Current health

- **Tests / lint:** none.
- **Last deploy:** N/A ã distribution model, not deployment. Repo has only one commit (`f962033 Initial: import existing ralph-station codebase`).
- **Open TODOs:** README says "Hardware design documentation will be available in `docs/` (coming soon)." That's the headline gap.
- **Bugs from read:** `ide/server.js` has a `safe()` path-traversal guard at line ~50 ã looks correct. Hono `cors()` is wide-open (`app.use('/api/*', cors())`); appropriate for a localhost appliance, **but** if any port forwarding is later enabled, this CORS becomes a problem. - **Security:**

  - WebSocket terminal at `/ws/terminal` exposes a full pty into `bash` ã same auth concern.
- **Performance:** Multi-arch Docker image is fine. `OLLAMA_KEEP_ALIVE: 24h` keeps the model hot; `OLLAMA_FLASH_ATTENTION: 1` is correct for RK3588 memory bandwidth.

### Shared work (do once, in `DuganLabs/.github` or `BaseNative`)

These items appear in 3-of-4 or 4-of-4 of the audited projects and should be solved at the org level.

1. **Reusable Terraform workflow.** Both `warrendugan/.github/workflows/infra.yml` and `warren-sys/.github/workflows/infra.yml` are verbatim 52-line copies (TF init against R2 ã validate ã plan ã apply on main). `greenput/.github/workflows/terraform.yml` is a richer staging/prod variant. Create `DuganLabs/.github/.github/workflows/tf-apply.yml@v1` (simple) + `tf-plan-apply-staged.yml@v1` (greenput-shaped). Migrate three repos.
2. **Reusable Cloudflare deploy workflow.** `warrendugan/deploy-www.yml` and `deploy-sys.yml` are duplicates with one path-filter swap. Plus `warren-sys/deploy.yml`. Plus `greenput`'s implicit deploy steps. Create `cf-worker-deploy.yml@v1` and `cf-pages-deploy.yml@v1` parameterised by `working-directory`, `paths`, `project-name`.
3. **dependabot.yml everywhere.** None of the four repos has `.github/dependabot.yml`. File a single template in `DuganLabs/.github/.github/dependabot.yml` that covers `npm`, `pip`, `terraform`, `docker`, `github-actions`. Apply org-wide via the `.github` repo's `default-community-files` mechanism.
4. **CodeQL / security scanning.** Same ã none configured. Add a `codeql.yml` reusable workflow.
5. **`bn prd init` (CLI work in BaseNative).** Three of four repos have explicit `bn prd init` placeholder TODOs in their `docs/PRD.md`. Land the CLI command and ask each repo to run it.
6. **PRD canonicalisation.** All four `docs/PRD.md` files are using the template structure but greenput has a *second* (richer) PRD at the repo root. Adopt one canonical location (`docs/PRD.md`) across the org and write an ADR (`duganlabs/docs/adr/0006-prd-location.md`).
7. **Adopt `@basenative/eslint-config` + `@basenative/tsconfig` everywhere.** Effort S/M each, risk low/medium. Single ADR (`0003` per duganlabs PRD M1 backlog) covers the policy; per-repo PRs do the work. Greenput is the harder one (Angular rules).
8. **Doppler bootstrap.** Per duganlabs PRD M1 ADR `0004`. None of the four repos use Doppler today; all use GH Action secrets. Migrate in one coordinated pass.
9. **Compatibility-date hygiene.** Greenput is the only one with a stale date (`2025-03-01`). The two warrendugan workers and warren-sys all use `2026-04-19`. Add a `bn doctor` lint that flags compatibility dates older than N months.
10. **Favicon preset is consistently adopted** ã three of four repos have an explicit `Favicon: unique @basenative/favicon preset for X` commit. Ralph-station has `favicon.svg` but no preset-commit-style history. Confirm or run.
